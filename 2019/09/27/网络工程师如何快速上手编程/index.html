我超讨厌编程如何快速上手编程

不少人说Alex啊，我就是为了逃避编程才做网络工程的啊，我看见代码就头疼，很多同学上网买关于python的书，我的天，跟砖头一样厚，别说是大家，我也肯定看不下去了，所以我们该怎样快速上手编程呢？

先掌握非常基础的Python技能

你必须要知道的knowledge~，就像你想学英语就要先认识26个英文字母一样。

  • Python的安装和运行
  • 写Python程序的工具或者软件
  • 能够简单通过Python定义一些简单的变量
  • 明白一些简单的逻辑判断和循环语句

  • Some Basic English, 用于阅读帮助文档

    把实际工作需求流程化，细节化，逻辑化

    编程能解决的问题就是那些流程化，细节化，逻辑化的东西，所以说仔细回忆一下，工作中都有哪些流程化，细节化，逻辑化的东西？

    • 流程化
      每次配置设备的时候，有哪些步骤是必须经历的，似乎有点像流水账一样的？有这种东西的话，那请你立马去休息，这种流程化的东西交给程序啊~
    • 细节化
      配置设备的时候，都配置了哪些设备，或者哪一组设备？他们的IP是多少？配置文件又存在哪？
    • 逻辑化
      你想做的配置有没有配置顺序的要求？如果有顺序请记下来。

    想零基础写代码？先拼积木！

    ？？？现在我啥也没学呢，就让我写代码？nonono,先要有最基础的Python知嗷。对于编程入门者来说，我们并不需要软件工程师那样特别好的理论基础和软件架构知识，Come on, Friends, we just use Programming as a tool, 我们只是把编程作为我们提升工作效率的工具而已，当然你要是有这种架构知识更好啦，我个人博客里有关于设计模式的知识，我们可以互相交流。

    那我们该如何搭建积木呢？

  • 百度，Google

    没错，百度，Google出来的程序多半是有一点小问题的，运行初期的错误怎么办？继续百度，Google, 这个时候你是带着问题去学习的，对知识的理解能力内化能力都是最好的，相反你看一遍砖头那么厚的Python书，你记住了吗？

    善于使用文档

    配置网络设备的时候，命令忘记了怎么办？肯定是打问号“？”，这个命令会告诉你有什么工具你还可以继续使用，Python呢？也有的哈，放心吧，而且Python的帮助信息更贴心嗷，这个在以后的文章会提到，先不赘述了。

总结

这一篇说实话，确实没啥干货，讲的大多数是方法，不过也是确确实实有效的方法。
